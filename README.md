# hidpi

CLI для включения настоящего HiDPI на физическом мониторе **без виртуального дисплея** и **без правки `/Library/Displays`**.

Использует приватные функции CoreGraphics/SkyLight (`CGSGetNumberOfDisplayModes`, `CGSGetDisplayModeDescriptionOfLength`, `CGSConfigureDisplayMode`), чтобы раскрыть и включить масштабированные (HiDPI, scale=2.0) режимы, которые macOS генерирует сама, но прячет из «Настроек».

Интерактивный выбор разрешения — в терминальном UI в стиле Pip-Boy: рамка, фосфорно-зелёная тема, стрелки для навигации.

## Установка одной командой

```sh
curl -fsSL https://raw.githubusercontent.com/titovcode/hidpi/main/install.sh | bash
```

Скрипт скачает исходники, соберёт релиз и положит бинарник в `/usr/local/bin/hidpi` (спросит пароль на `sudo`). Требуется macOS и Xcode Command Line Tools (`xcode-select --install`). После установки просто запусти `hidpi`.

## Сборка из исходников

```sh
git clone https://github.com/titovcode/hidpi.git
cd hidpi
swift build -c release
# бинарник: .build/release/hidpi
```

## Использование

Проще всего — интерактивно. Запусти без аргументов, выбери разрешение из списка по номеру:

```sh
hidpi                            # список HiDPI-разрешений → выбор по номеру
hidpi pick --all                 # то же, но показать вообще все режимы (включая 1x и мелкие)
```

Список отсортирован от максимального рабочего места к меньшему; крупнейший HiDPI помечен `★ макс. места`, текущий — `← current`. Если у разрешения несколько частот, спросит какую (по умолчанию — максимальная).

Неинтерактивные команды (для скриптов/автозапуска):

```sh
hidpi list                       # онлайн-дисплеи (индекс, текущий режим)
hidpi modes -d 0 --hidpi         # только HiDPI-режимы дисплея 0
hidpi set -d 0 -w 2560 -h 1440 -s 2   # макс. места + HiDPI
hidpi set -d 0 -w 2560 -h 1440 -s 1   # нативный, без HiDPI
hidpi set -d 0 --mode 168        # по индексу из `modes`
hidpi install                    # скопировать бинарник в /usr/local/bin (спросит sudo)
```

Опции: `-d/--display`, `-w/--width`, `-h/--height`, `-s/--scale` (по умолчанию 2.0), `-b/--bits` (16|32), `--mode IDX`, `--hidpi` (для `modes`), `-a/--all` (для `pick`).

### Какой режим выбирать

`scale=2.0` означает, что рабочее пространство W×H отрисовывается в 2W×2H пикселей.
- На **4K-панели** режим `1920x1080 HiDPI` даёт чёткий «ретина»-2x → рендер 3840×2160 в нативные пиксели.
- На **1440p-панели** идеальный 2x даёт только `1280x720 HiDPI` (крупный UI). `1920x1080 HiDPI` будет рендериться в 3840×2160 и даунскейлиться в 1440p (суперсэмплинг — мягко, нагружает GPU).

## Границы метода

Рантайм-API только **раскрывает** HiDPI-режимы, которые macOS уже сгенерировала. Если EDID монитора вынуждает macOS вообще не создавать HiDPI (`hidpi modes --hidpi` пусто), рантайм не создаст режим из ничего — тогда нужен EDID-override в `/Library/Displays`.

## Дисклеймер

Приватный недокументированный API. Не совместим с App Store, может ломаться между версиями macOS. Layout структуры режима (`modes_D4`) взят из проекта [RDM](https://github.com/usr-sse2/RDM) (MIT) и проверен на macOS 15.7 / Apple Silicon. `set` меняет активный дисплей немедленно; выбор сохраняется между перезагрузками (`kCGConfigurePermanently`). `sudo` не требуется.
